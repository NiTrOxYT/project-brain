PBS-0002

Workspace Specification

Status: Draft

Version: 1.0.0

Depends On:
PBS-0001

⸻

Purpose

This specification defines the .brain workspace.

The workspace is the persistent knowledge database maintained by Project Brain.

The runtime reads and writes the workspace.

The workspace never contains executable code.

⸻

Root Structure

.brain/

manifest.json

knowledge/

graph/

index/

cache/

history/

state/

⸻

Workspace Rules

The workspace is the single source of truth for project intelligence.

Every file inside the workspace must have one responsibility.

Files are appendable whenever practical.

Generated data must be deterministic.

Temporary runtime data belongs only inside cache/.

No package may store runtime state outside the workspace.

⸻

Directories

knowledge/

Persistent structured project knowledge.

Contains information that represents the logical understanding of the project.

Examples:

project.json

architecture.json

modules.json

patterns.json

services.json

routes.json

⸻

graph/

Stores relationships.

Examples:

nodes.json

edges.json

dependencies.json

ownership.json

⸻

index/

Stores deterministic indexes.

Examples:

files.json

symbols.json

imports.json

exports.json

hashes.json

⸻

cache/

Temporary runtime data.

May be deleted without losing project knowledge.

Examples:

working-set.json

recent-context.json

last-plan.json

⸻

history/

Append-only historical records.

Examples:

tasks.json

reviews.json

changes.json

⸻

state/

Current runtime state.

Examples:

session.json

lock.json

runtime.json

⸻

Responsibilities

knowledge/
stores meaning.

graph/
stores relationships.

index/
stores facts.

cache/
stores temporary information.

history/
stores permanent history.

state/
stores runtime state.
