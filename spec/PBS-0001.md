PBS-0001

Project Brain Architecture Specification

Status: Draft

Version: 1.0.0

Authors:
Project Brain Team

⸻

1. Purpose

This specification defines the architecture of Project Brain.

All implementations MUST conform to this document.

Any future architectural changes MUST be introduced through a new PBS specification rather than modifying this document.

⸻

2. Goals

Project Brain is a persistent AI runtime for software engineering.

The runtime exists to provide long-term project intelligence while minimizing unnecessary context retrieval.

Project Brain MUST remain independent from any specific LLM or coding agent.

⸻

3. Design Principles

The following principles are immutable.

1. Single Responsibility

Every package has one responsibility.

Every service has one responsibility.

Every JSON file has one responsibility.

⸻

2. Machine First

Project Brain stores structured knowledge.

Human documentation is generated from structured knowledge.

Structured data is always the source of truth.

⸻

3. Runtime Independence

Project Brain MUST NOT depend on GPT, Claude, Gemini, Antigravity or any other model.

AI integrations exist only as adapters.

⸻

4. Workspace Independence

Every project contains a .brain workspace.

The runtime never stores executable code inside .brain.

The workspace only stores project data.

⸻

5. Deterministic Execution

Every operation must produce deterministic results whenever possible.

⸻

6. Incremental Synchronization

The runtime never rebuilds the workspace unless explicitly requested.

Only affected data may be updated.

⸻

7. Explicit Contracts

Every public API must be documented.

Every package must expose a stable interface.

Internal implementations may change.

Public contracts may not.

⸻

8. Versioned Architecture

Breaking architectural changes require a new PBS specification.

The runtime must remain backwards compatible whenever practical.

⸻

9. Plugin Architecture

Language support, framework support and AI integrations are plugins.

They are never hardcoded into the kernel.

⸻

10. Runtime Before AI

The runtime owns project intelligence.

AI consumes project intelligence.

The runtime never depends on AI.
