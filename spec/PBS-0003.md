PBS-0003

Runtime Services

Status: Draft

Version: 1.0.0

Depends On:
PBS-0001
PBS-0002

⸻

Purpose

This specification defines the runtime services that compose Project Brain.

Every runtime capability is implemented as a service.

Services communicate through public contracts only.

Services never access each other’s internal implementation.

⸻

Service Lifecycle

Every runtime service follows the same lifecycle.

initialize()

↓

load()

↓

execute()

↓

sync()

↓

dispose()

No service may define a different lifecycle.

⸻

Core Services

Owns the workspace manifest.

Responsibilities

Load manifest

Validate manifest

Save manifest

Upgrade manifest

⸻

WorkspaceService

Owns the .brain workspace.

Responsibilities

Create workspace

Verify workspace

Repair workspace

Delete workspace

⸻

IndexService

Responsibilities

Index files

Index symbols

Index imports

Index exports

Detect changes

⸻

KnowledgeService

Responsibilities

Generate knowledge

Update knowledge

Read knowledge

Validate knowledge

⸻

GraphService

Owns project relationships.

Responsibilities

Generate graph

Update graph

Resolve dependencies

Query graph

⸻

Responsibilities

⸻

⸻

Reviews

Events

⸻

Communication Rules

Services never communicate directly.

All communication passes through the runtime.

No service may mutate another service’s data.

Each service owns its own domain.

⸻

Design Principle

Services expose stable interfaces.

Implementations remain private.

Only public contracts are considered part of the runtime API.
